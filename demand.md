# 投标文件审核系统需求文档

## 系统概述
设计一个智能化的投标文件审核系统，能够自动分析需求文档和投标文件，生成审核任务并进行智能评估。

## 界面布局

### 三栏式布局
1. **左侧（1/4）**：文档输入区域
2. **中间（2/5）**：任务列表
3. **右侧（2/5）**：任务审核详情

### 左侧 - 文档输入区域
- **需求文档输入**
  - 支持直接粘贴文本内容
  - 可编辑的文本框（contenteditable）
  - 字符数实时显示
  - 清空按钮
  - 格式要求：Markdown格式支持

- **投标文件输入**
  - 支持直接粘贴文本内容
  - 可编辑的文本框（contenteditable）
  - 字符数实时显示
  - 清空按钮
  - 格式要求：Markdown格式支持

- **分析按钮**
  - 触发文档分析和任务生成
  - 加载状态显示
  - 禁用状态控制

### 中间 - 任务列表
- **任务列表展示**
  - 任务数量统计
  - 任务卡片布局
  - 每个任务显示：
    - 任务ID
    - 任务标题
    - 任务描述（截断显示）
    - 任务状态（待审核/已审核）

- **任务交互**
  - 点击任务卡片选中任务
  - 选中状态高亮显示
  - 悬停效果

### 右侧 - 任务审核详情
- **任务信息展示**
  - 任务标题
  - 审核状态

- **审核结果**
  1. **审核结论**
     - 状态标签：【通过】【不通过】【待确认】
     - 不同状态使用不同颜色标识
     - 重新审核按钮

  2. **审核原因**
     - 详细文字说明
     - 支持富文本格式

  3. **来源信息**
     - 需求来源（具体章节位置）
     - 投标来源（具体章节位置）

  4. **评分详情**（可选）
     - 各维度评分
     - 进度条可视化展示

## 核心功能流程

### 1. 文档输入与分析
```
用户输入信息 → 用户输入投标文件 → 点击"开始分析" →
调用LLM API → 分析并生成任务列表 → 更新任务列表
```

### 2. 任务审核流程
```
用户选择任务 → 点击"开始审核" →
调用LLM API进行审核 → 显示审核结果 → 可重新审核
```

## 技术实现要求

### 前端技术栈
- **Vue 3**：框架选择
- **Tailwind CSS**：样式框架
- **Axios**：HTTP请求库

### 数据结构设计

#### 任务（Task）
```javascript
{
  id: number,
  title: string,           // 任务标题
  description: string,     // 任务描述
  requirementSource: string, // 需求来源
  bidSource: string,      // 投标来源
  review: {               // 审核结果（可选）
    status: string,       // '通过' | '不通过' | '待确认'
    reason: string,       // 审核原因
    score: {              // 各维度评分（可选）
      [key: string]: number
    }
  }
}
```

#### 审核结果（Review）
```javascript
{
  status: '通过' | '不通过' | '待确认',
  reason: string,
  requirementSource: string,
  bidSource: string,
  score?: {
    [dimension: string]: number // 0-100
  }
}
```

### API接口设计

#### 1. 分析接口（生成任务）
```javascript
POST /api/analyze
Content-Type: application/json

{
  requirement: string,
  bid: string
}

Response:
{
  tasks: Task[]
}
```

#### 2. 审核接口
```javascript
POST /api/review
Content-Type: application/json

{
  taskId: number,
  requirement: string,
  bid: string
}

Response:
{
  review: Review
}
```

## 界面设计规范

### 颜色主题
- **主色调**：蓝色系（#1e40af）
- **成功色**：绿色（#16a34a）
- **错误色**：红色（#dc2626）
- **警告色**：黄色（#f59e0b）
- **中性色**：灰色系（#f3f4f6 - #111827）

### 字体
- **中文字体**：Noto Sans SC
- **英文字体**：Inter
- **字体粗细**：400, 500, 600, 700

### 交互状态
- **加载状态**：旋转加载动画
- **禁用状态**：灰色背景，指针不可点击
- **悬停效果**：阴影增强，微位移
- **选中状态**：边框高亮，背景色变化
