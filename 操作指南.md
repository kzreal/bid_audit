# 投标文件审核系统 - 操作指南

## 服务架构

```
浏览器 (同事访问)
     ↓
前端 (Vite Dev Server: 5173)
     ↓ Vite 代理
后端 (Flask Server: 8888)
     ↓
HiAgent API (远程 AI 服务)
```

**说明：**
- 前端通过 Vite 代理将 `/hiagent` 和 `/health` 请求转发到本地后端
- 后端作为中间层连接到远程 HiAgent API

## 端口说明

| 端口 | 服务 | 说明 |
|------|------|------|
| 5173 | 前端服务 | Vite 开发服务器默认端口，浏览器访问地址 |
| 8888 | 后端服务 | Flask API 服务器，前端调用接口地址 |

## 启动服务

### 方法一：分别启动

#### 启动后端
```bash
# 确保在项目根目录
cd /Users/kyle/projects/投标文件审核

# 启动后端服务（后台运行）
python3 backend_server.py > backend.log 2>&1 &

# 等待几秒后检查状态
sleep 3
lsof -i :8888 -t && echo "✅ 后端服务启动成功"
```

#### 启动前端
```bash
# 进入前端目录
cd bid-review-system

# 启动前端服务（后台运行）
npm run dev > frontend.log 2>&1 &

# 等待几秒后检查状态
sleep 5
lsof -i :5173 -t && echo "✅ 前端服务启动成功"
```

### 方法二：一键启动

创建启动脚本 `start.sh`：
```bash
#!/bin/bash

echo "=== 启动投标文件审核系统 ==="

# 进入项目目录
cd "$(dirname "$0")"

# 停止已存在的服务
echo "停止现有服务..."
pkill -f backend_server.py 2>/dev/null
pkill -f vite 2>/dev/null
sleep 2

# 启动后端
echo "启动后端服务..."
python3 backend_server.py > backend.log 2>&1 &
sleep 3
if lsof -i :8888 -t 2>/dev/null; then
    echo "✅ 后端服务启动成功 (端口 8888)"
else
    echo "❌ 后端服务启动失败"
    cat backend.log | tail -20
    exit 1
fi

# 启动前端
echo "启动前端服务..."
cd bid-review-system
npm run dev > frontend.log 2>&1 &
sleep 5
if lsof -i :5173 -t 2>/dev/null; then
    echo "✅ 前端服务启动成功 (端口 5173)"
else
    echo "❌ 前端服务启动失败"
    cat frontend.log | tail -20
    exit 1
fi

echo ""
echo "=== 系统启动完成 ==="
echo "前端访问地址: http://localhost:5173"
echo "后端 API 地址: http://127.0.0.1:8888"
```

使用方法：
```bash
chmod +x start.sh
./start.sh
```

## 常见问题

### Q1: 同事访问 `http://10.128.228.194:5173` 时后端连接失败？

**可能原因：**
1. 后端服务未启动
2. 后端配置修改后未重启

**检查方法：**
```bash
# 1. 检查后端是否运行
lsof -i :8888 -t

# 2. 测试后端健康检查
curl http://localhost:8888/health

# 3. 查看后端日志
tail -50 backend.log
```

**解决方法：**
1. 如果后端未运行，启动后端：`python3 backend_server.py`
2. 如果后端已运行但修改了配置，需要重启：
   ```bash
   pkill -f backend_server.py
   python3 backend_server.py
   ```

### Q2: 前端启动在 5174/5175 端口，不是 5173？

**原因：** 5173 端口被占用，Vite 自动寻找可用端口。

**解决方法：**
1. 关闭占用端口的进程：`pkill -f vite`
2. 重新启动：`npm run dev`

**访问地址：** 使用实际启动的端口，如 `http://10.128.228.194:5174`

### Q3: 修改了 `vite.config.js` 后前端仍然使用旧配置？

**原因：** Vite 配置文件修改后需要重启开发服务器才能生效。

**解决方法：**
```bash
# 停止前端
pkill -f vite

# 重新启动
cd bid-review-system
npm run dev
```

### Q4: 同事无法访问前端页面？

**可能原因：**
1. 防火墙阻止了 5173 端口
2. 前端没有监听在 `0.0.0.0`
3. IP 地址不正确

**检查方法：**
```bash
# 1. 检查前端监听地址
lsof -i :5173 -n | grep LISTEN

# 2. 查看当前机器的 IP 地址
ifconfig | grep "inet "

# 3. 测试本地是否可访问
curl http://localhost:5173
```

**解决方法：**
1. 确保 `vite.config.js` 中 `host` 设置为 `0.0.0.0`
2. 配置防火墙允许 5173 和 8888 端口
3. 确认提供给同事的 IP 地址正确

### Q5: 端口被占用？

**查看占用进程：**
```bash
lsof -i :5173 -t  # 查看前端端口
lsof -i :8888 -t  # 查看后端端口
```

**强制关闭：**
```bash
pkill -9 -f vite         # 强制关闭前端
pkill -9 -f backend_server  # 强制关闭后端
```

### Q4: 如何修改端口号？

**前端端口：** 修改 `vite.config.js`
```javascript
server: {
  port: 5173  // 改为其他端口
}
```

**后端端口：** 修改 `backend_server.py`
```python
if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8888, debug=True)  // 改为其他端口
```

**注意：** 修改端口后，需要更新前端的 `VITE_API_BASE_URL` 配置。
