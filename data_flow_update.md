# 单切片审核节点的输出数据结构（JSON）：

```json
{
  "output": [
    {
      "suggestion": "该切片的审核建议",
      "evidence": "行号列表，或字符串 \"null\"（表示本切片与任务无关）"
    }
  ]
}
```

示例：
```json
{
  "output": [
    {
      "evidence": "4",
      "suggestion": "本切片中招标编号共出现1处（第4行），实际内容为'AFDECVRPSMX2438954'，与标准值'ADBVDJSNVCSOIJCV'不一致。具体差异：标准值以'ADB'开头，实际内容以'AFD'开头；标准值全为大写字母，实际内容包含数字'2438954'。"
    }
  ]
}
```

# 总结节点

## 输入 JSON

```json
{
  "context": {
    "task": "单条审核任务描述",
    "reviews": [
      {
        "title": "该切片所属章节名称",
        "suggestion": "该切片的审核建议",
        "evidence": "行号列表，或字符串 \"null\"（表示本切片与任务无关）"
      }
    ]
  }
}
```

## 输出 JSON

```json
{
  "results": [
    {
      "conclusion": "通过 / 不通过 / 待确认",
      "reasons": [
        {
          "reason": "在"章节名称"中，……（完整描述该切片的所有发现）",
          "evidence": "276, 684-713, 758-770"
        }
      ]
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `results` | `Arr<Obj>` | 每次输出恰好包含一个对象元素 |
| `conclusion` | `Str` | 固定为 `"通过"`、`"不通过"` 或 `"待确认"` 之一 |
| `reasons` | `Arr<Obj>` | 每个有效切片对应一个对象元素，无关切片不输出 |
| `reason` | `Str` | 以 `"在"{title}"中，……"` 格式引用章节，完整描述该切片所有发现 |
| `evidence` | `Str` | 该切片行号按升序排列，连续行用连字符表示；无内容时固定输出字符串 `"null"` |
