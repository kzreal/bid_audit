## Design Context

### Users
投标专员，在投标截止前紧急审核文件。时间压力大，需要快速定位问题、高效完成任务。他们每天处理多份投标文件，熟悉业务流程但对技术不敏感，界面必须减少认知负担和操作步骤。

### Brand Personality
**专业、高效、克制**
- Voice: 安静的专业感，用清晰的操作引导代替华丽动画
- Tone: 工具化、中性、不抢戏
- Emotional goal: 让用户感到"这个工具不会拖我后腿"

### Aesthetic Direction
- **Visual tone**: Vercel 极简风格 — 大量留白，1px 细边框，极小圆角（2px）
- **Primary reference**: Vercel Dashboard — 单色为主，蓝色仅用于关键操作
- **Anti-reference**: 不要花哨渐变、不要大面积彩色、不要多余装饰元素
- **Theme**: 浅色模式（Light mode only）
- **Colors**: Vercel 蓝 (#0070f3) 作为唯一强调色，状态色仅用于审核结论

### Design Principles

1. **减少步骤，合并流程** — 用户时间紧迫，相关操作应合并到同一视图（如 Tab 合并）
2. **状态清晰，反馈即时** — 每个操作都应有明确的进度指示（spinner、进度条、状态文字）
3. **视觉层级分明** — 标题 18px/14px 两级，正文 14px，辅助 12px。用字重和颜色区分，不要用大小
4. **克制用色** — 黑白灰为主，蓝色仅用于可点击和 active 状态，红/绿/黄仅用于审核结论
5. **信息密度优先** — 投标专员需要看到足够信息做判断，但不要堆砌。关键信息突出，次要信息折叠
