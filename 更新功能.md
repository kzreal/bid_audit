## 一、整体布局

系统分为左右两栏：

- **左侧操作区（40%宽）**：包含四个 Tab，负责所有操作交互
- **右侧预览区（60%宽）**：负责 Word 文档的原生渲染与高亮跳转

│  投标文件审核系统                                              │
├──────────────────────────┬──────────────────────────────────┤
│     左侧操作区 (40%)      │        右侧预览区 (60%)           │
│  ┌────┬────┬────┬────┐   │                                   │
│  │创建│创建│任务│审核│   │   Word 文档原生渲染                 │

│  [Tab 内容区]             │                                   │
└──────────────────────────┴──────────────────────────────────┘

## 二、左侧操作区

### Tab 1 · 创建项目

**功能目标：** 上传 Word 文件，解析为 Markdown 并切片，创建项目。

**界面布局：**

┌─────────────────────────────┐
│  📁 上传 Word 文件           │
│  ┌─────────────────────┐    │
│  │  拖拽或点击上传 .docx │    │
│  └─────────────────────┘    │
│                              │
│  📄 解析设置                 │
│  切片层级：零级 一级 二级 三级 全部│
│  解析消息通知      [ 下载切片 ]  │
│                              │
│  项目名称: [____________]    │
│                              │
│               [ 创建项目 ] │
└─────────────────────────────┘

复制

**交互逻辑：**

- 上传文件后 → 右侧预览区**实时渲染** Word 原文内容
- 点击层级（零级、一级等）开始将word转化为markdown，成功后在解析消息通知处显示解析完成
- 点击「创建项目」→ 保存项目信息，自动跳转至 Tab 2（创建任务）

## 三、右侧预览区

### 3.1 基础功能

- 上传 Word 文件后，使用 `mammoth.js` 将 `.docx` 转换为 HTML 并渲染
- 支持**鼠标滚轮**翻滚浏览全文
- 顶部工具栏提供**预览模式切换**：
  - `原文模式`：渲染 Word 原始样式
  - `切片视图`：不同切片用不同背景色区分，显示切片编号与边界线
- 顶部提供**全文搜索框**，支持关键词高亮定位

**界面布局：**

┌──────────────────────────────────────┐
│  预览模式: [原文 | 切片视图]  🔍搜索  │
│  ────────────────────────────────    │
│  第1行  投标文件                      │
│  第2行  项目名称：XX工程              │
│  ...                                 │
│ ┌────────────────────────────────┐   │
│ │ 第23行 报价总额：150万元        │   │  ← 高亮行
│ └────────────────────────────────┘   │

复制

### 3.2 高亮跳转交互

点击 Tab 4 审核结果中的「📍 投标来源 → 第N行」后，触发以下流程：

1. 右侧预览区**平滑滚动**至目标行（`scrollIntoView` + smooth behavior）
2. 目标行呈现**高亮背景**（推荐橙黄色）+ 左侧竖线标注
3. 高亮效果持续 **3 秒后渐隐**，或保持至下一次点击
4. 若当前为切片视图，自动切换回原文模式再执行跳转

## 四、技术实现建议

| 功能模块 | 推荐方案 |
|---|---|
| Word 渲染 | `mammoth.js` 转 HTML，保留原始样式 |
| 行号注入 | 渲染时为每个段落注入 `data-line="N"` 属性 |
| 高亮跳转 | `element.scrollIntoView({ behavior: 'smooth' })` + CSS highlight class |
| 切片视图 | 不同切片用不同背景色区分，边界加分隔线 |
| Markdown 解析 | `marked.js` 或 `unified` 生态 |
| LLM 任务转换 | 前端调用后端接口，流式返回解析结果 |
| 状态管理 | React 项目推荐 Zustand，Vue 项目推荐 Pinia |

---

## 五、建议开发顺序

1. **搭建布局骨架** — 左右分栏 + 四个 Tab 框架，确定组件结构
2. **实现 Word 预览** — mammoth.js 渲染 + 行号 data 属性注入
3. **实现高亮跳转** — 最核心的交互体验，优先打通
4. **实现任务管理** — Tab 3 的增删改排序与状态流转
5. **接入 LLM 转换** — Tab 2 的自然语言 → 结构化任务解析
6. **打通完整审核流程** — Tab 1 → Tab 2 → Tab 3 → Tab 4 全链路联调
