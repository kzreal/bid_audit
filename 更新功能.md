## 二、数据存储（SQLite）

### 2.1 数据库设计

使用 SQLite 数据库存储所有项目数据，数据库文件：`data/bid_review.db`

**数据库表结构：**

```sql
-- 项目表
CREATE TABLE projects (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    file_name TEXT,
    file_size INTEGER,
    slice_level INTEGER DEFAULT 0,
    slice_count INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    last_opened_at TEXT
);

-- Word 文件表（存储二进制文件）
CREATE TABLE word_files (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    file_data BLOB NOT NULL,
    file_name TEXT NOT NULL,
    created_at TEXT NOT NULL,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 切片表
CREATE TABLE slices (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    index_num INTEGER NOT NULL,
    title TEXT,
    level INTEGER DEFAULT 0,
    content TEXT,
    start_line INTEGER,
    end_line INTEGER,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 审核任务表
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    title TEXT NOT NULL,
    description TEXT,
    status TEXT DEFAULT 'pending',
    sort_order INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 审核结果表
CREATE TABLE reviews (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    project_id TEXT NOT NULL,
    conclusion TEXT,
    reason TEXT,
    evidence TEXT,
    bid_source TEXT,
    requirement_source TEXT,
    slices_reviews TEXT,
    created_at TEXT NOT NULL,
    FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);
```

**ER 关系图：**

```
projects (1) ─────< (N) word_files
    │
    │
    └────< (N) slices
    │
    │
    └────< (N) tasks ─────< (N) reviews
```

### 2.2 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/projects` | GET | 获取所有项目列表 |
| `/api/projects` | POST | 创建新项目 |
| `/api/projects/:id` | GET | 获取项目详情（含切片、任务、审核结果） |
| `/api/projects/:id` | PUT | 更新项目信息 |
| `/api/projects/:id` | DELETE | 删除项目 |
| `/api/projects/:id/word` | GET | 下载项目 Word 文件 |
| `/api/projects/:id/slices` | GET | 获取项目切片列表 |
| `/api/projects/:id/tasks` | GET/POST | 获取/添加任务 |
| `/api/projects/:id/tasks/:taskId/review` | POST | 提交审核结果 |

### 2.3 核心功能

1. **项目创建** → 保存项目信息 + Word 文件 + 切片数据
2. **项目加载** → 从数据库读取 Word 文件渲染 + 切片数据给 HiAgent
3. **任务管理** → 创建/编辑/删除任务
4. **审核结果** → 保存每次审核结果

## 三、左侧操作区

### Tab 1 · 创建项目

**功能目标：** 上传 Word 文件，解析为 Markdown 并切片，创建项目。

**界面布局：**

**交互逻辑：**

- 顶部**项目选择器**：下拉选择历史项目，选择后自动加载 Word 预览和切片数据
- 上传文件后 → 右侧预览区**实时渲染** Word 原文内容
- 点击**开始解析** → 将 Word 转化为 Markdown，成功后显示解析完成
- 点击「创建项目」→ 保存到数据库，自动跳转至 Tab 2（创建任务）

**项目选择器交互：**
- 选择历史项目 → 从数据库加载 Word 文件 + 切片数据
- 切片数据直接传给 HiAgent 用于审核
- 支持删除历史项目

## 四、右侧预览区

### 4.1 基础功能

- 上传 Word 文件后，使用 `docx-preview` 将 `.docx` 转换为 HTML 并渲染
- 支持**鼠标滚轮**翻滚浏览全文
- 顶部工具栏提供**预览模式切换**：
  - `原文模式`：渲染 Word 原始样式
  - `切片视图`：不同切片用不同背景色区分，显示切片编号与边界线
- 顶部提供**全文搜索框**，支持关键词高亮定位

**界面布局：**

┌──────────────────────────────────────┐
│  预览模式: [原文 | 切片视图]  🔍搜索  │
│  ────────────────────────────────    │
│  第1行  投标文件                      │
│  第2行  项目名称：XX工程              │
│  ...                                 │
│ ┌────────────────────────────────┐   │
│ │ 第23行 报价总额：150万元        │   │  ← 高亮行
│ └────────────────────────────────┘   │
│  第24行 付款方式：分期付款            │
│  ...                                 │
└──────────────────────────────────────┘

---

### 4.2 高亮跳转交互

点击 Tab 4 审核结果中的「📍 投标来源 → 第N行」后，触发以下流程：

1. 右侧预览区**平滑滚动**至目标行（`scrollIntoView` + smooth behavior）
2. 目标行呈现**高亮背景**（推荐橙黄色）+ 左侧竖线标注
3. 高亮效果持续 **3 秒后渐隐**，或保持至下一次点击
4. 若当前为切片视图，自动切换回原文模式再执行跳转

## 五、技术实现建议

| 功能模块 | 推荐方案 |
|---|---|
| 数据库 | SQLite，文件存储 `data/bid_review.db` |
| Word 渲染 | `docx-preview` 转 HTML，保留原始样式 |
| 行号注入 | 渲染时为每个段落注入 `data-line="N"` 属性 |
| 高亮跳转 | `element.scrollIntoView({ behavior: 'smooth' })` + CSS highlight class |
| 切片视图 | 不同切片用不同背景色区分，边界加分隔线 |
| 后端框架 | Python Flask（复用现有后端） |
| 状态管理 | Vue 项目使用 Pinia |

## 六、数据加载流程

### 6.1 选择历史项目

```
用户选择项目
    │
    ▼
前端请求 GET /api/projects/:id
    │
    ├──► 返回项目信息（名称、创建时间等）
    ├──► 返回切片列表（用于 HiAgent 审核）
    └──► 返回 Word 文件流（用于右侧预览）

前端渲染 Word 文件到右侧预览区
前端将切片数据存入 store（用于审核时发送给 HiAgent）
```

### 6.2 审核流程（使用数据库中的切片数据）

```
用户点击"开始审核"
    │
    ▼
从 store 获取当前项目的切片数据
    │
    ▼
调用 HiAgent API（slices_content 来自数据库）
    │
    ▼
审核结果保存到数据库
    │
    ▼
更新任务状态
```

## 七、建议开发顺序

1. **搭建 SQLite 数据库** — 创建表结构，实现基础 CRUD 接口
2. **改造后端接口** — 将项目/任务/审核结果存储到数据库
3. **搭建布局骨架** — 左右分栏 + 四个 Tab 框架，确定组件结构
4. **实现项目选择器** — Tab 1 增加项目下拉选择，加载历史数据
5. **实现 Word 预览** — docx-preview 渲染 + 行号 data 属性注入
6. **实现高亮跳转** — 最核心的交互体验，优先打通
7. **实现任务管理** — Tab 3 的增删改排序与状态流转
8. **接入 LLM 转换** — Tab 2 的自然语言 → 结构化任务解析
9. **打通完整审核流程** — Tab 1 → Tab 2 → Tab 3 → Tab 4 全链路联调
